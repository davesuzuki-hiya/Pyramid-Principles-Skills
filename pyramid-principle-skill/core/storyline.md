# Building Compelling Storylines

A storyline transforms analysis into narrative. It's how you take a pyramid of logic and present it in a way that moves people to action.

## The Governing Thought

Every presentation/document needs one Governing Thought - the single most important message you want your audience to remember.

### Characteristics of a Strong Governing Thought

- **Singular**: One main message (not a list)
- **Actionable**: Points to what should happen next
- **Surprising**: Not obvious before your analysis
- **Memorable**: Can be stated in one sentence

### Examples

**Weak Governing Thought:**
> "This report covers Q3 performance across our product lines."
- Problem: Descriptive, not actionable, not memorable

**Strong Governing Thought:**
> "We must pivot to usage-based pricing within 6 months to capture the $40M SMB opportunity before Competitor X establishes market dominance."
- Singular: One strategic recommendation
- Actionable: Clear next step (pivot pricing)
- Surprising: Specific timeline and competitive urgency
- Memorable: Stakes are clear

## The Rule of Three

Support your Governing Thought with exactly 3 key arguments. Not 2 (feels incomplete), not 4+ (too many to remember).

```
GOVERNING THOUGHT
"We should enter the UK market first"

├── KEY ARGUMENT 1
│   "UK has the highest revenue potential"
│
├── KEY ARGUMENT 2
│   "UK has the lowest entry barriers"
│
└── KEY ARGUMENT 3
    "UK enables future European expansion"
```

### Why Three?

- **Cognitive load**: People remember 3 things easily
- **Completeness**: 3 feels like a complete argument
- **Credibility**: Shows you've thought it through, but prioritized

If you have more than 3 arguments, either:
1. Group them under 3 themes
2. Prioritize and save the rest for Q&A
3. Move supporting points to sub-levels

## Storyline Structures

### Structure 1: Recommendation-First (Most Common)

Best for: Executives, time-constrained audiences, action-oriented contexts

```
1. SCQA Introduction (2 min)
   - Situation, Complication, Question, Answer

2. Governing Thought (30 sec)
   - State your main recommendation clearly

3. Key Argument 1 + Evidence (3-5 min)
4. Key Argument 2 + Evidence (3-5 min)
5. Key Argument 3 + Evidence (3-5 min)

6. Next Steps / Call to Action (1-2 min)

[Appendix: Supporting details, methodology, additional data]
```

### Structure 2: Situation → Analysis → Recommendation

Best for: Complex problems requiring audience buy-in, skeptical audiences

```
1. Situation Overview (3-5 min)
   - Current state, key metrics, context

2. Analysis Deep Dive (10-15 min)
   - What we examined
   - What we found
   - What it means

3. Recommendation (3-5 min)
   - Based on analysis, we recommend...
   - Supported by three key arguments

4. Implementation Plan (3-5 min)
   - Timeline, resources, milestones

5. Risks and Mitigations (2-3 min)
```

### Structure 3: Options Framework

Best for: Decision-making meetings, when stakeholders need to choose

```
1. Decision Context (2-3 min)
   - SCQA: What decision do we face?

2. Evaluation Criteria (2-3 min)
   - What matters? How will we judge options?

3. Option A: Description + Pros/Cons (3-5 min)
4. Option B: Description + Pros/Cons (3-5 min)
5. Option C: Description + Pros/Cons (3-5 min)

6. Comparative Analysis (3-5 min)
   - Side-by-side on criteria

7. Recommendation (2-3 min)
   - Our recommended option and why
```

## Horizontal Logic: Connecting Arguments

Your 3 key arguments must relate to each other in a logical way:

### Deductive Flow (If A, then B, then C)
Arguments build on each other in sequence.

```
1. The market is shifting to mobile-first
2. Our product is optimized for desktop
3. Therefore, we must rebuild for mobile

(Each argument depends on the previous)
```

### Inductive Flow (A, B, and C all point to X)
Independent arguments that together support the conclusion.

```
1. Revenue is growing 40% YoY
2. Churn is at historic lows
3. NPS is industry-leading

→ Therefore, now is the right time to raise prices
(Each argument is independent evidence)
```

### Comparative Flow (Evaluate X vs Y vs Z)
Arguments compare options against criteria.

```
1. On revenue potential: UK > Germany > France
2. On entry barriers: UK < Germany < France
3. On strategic value: UK > Germany = France

→ Therefore, UK is the best market to enter
```

## Crafting the Narrative Arc

### The Setup (SCQA)
Hook the audience in the first 60 seconds:
- Ground them in familiar territory (Situation)
- Create tension (Complication)
- Make them want the answer (Question)
- Deliver your thesis (Answer)

### The Build (Key Arguments)
Each argument should:
- Open with the argument stated clearly (not revealed at the end)
- Provide evidence that proves the argument
- Conclude with the "so what" for this section
- Transition smoothly to the next argument

### The Climax (Synthesis)
Bring the arguments together:
- "Taken together, these three factors indicate..."
- "This convergence of evidence supports..."
- Restate the Governing Thought with added weight

### The Resolution (Call to Action)
Make it crystal clear what happens next:
- Specific next steps
- Who is responsible
- By when
- What success looks like

## Storytelling Techniques

### 1. The Contrast Pattern
Set up what IS versus what COULD BE:

> "Today, we convert 0.8% of trials. Industry leaders convert 3-5%. The gap represents $6M in unrealized annual revenue. Here's how we close it."

### 2. The Zoom Pattern
Start wide, then narrow:

> "The global call security market is $4B. The SMB segment is $800M and growing 25% YoY. Within SMB, the 'prosumer' segment—small business owners using personal phones—is $200M and completely underserved. That's our target."

### 3. The Unexpected Insight
Lead with something surprising:

> "We assumed enterprise was our most profitable segment. We were wrong. SMB customers have 40% higher LTV when accounting for expansion revenue and lower support costs."

### 4. The Stakes Elevation
Make the audience feel the urgency:

> "If we don't act in the next 6 months, Competitor X will lock up the UK market. Their funding round specifically targets European expansion. The window is closing."

## Storyline Quality Checklist

- [ ] Governing Thought is singular, actionable, memorable
- [ ] Exactly 3 key arguments (not 2, not 4+)
- [ ] Each argument can stand alone but strengthens with others
- [ ] SCQA opening hooks in <60 seconds
- [ ] Evidence supports each argument (not just asserted)
- [ ] Transitions between arguments are smooth
- [ ] Clear call to action with specific next steps
- [ ] Audience needs are addressed (what do THEY care about?)
- [ ] Appendix catches supporting detail (keeps main flow clean)

## Example: Complete Storyline

### Governing Thought
> "Hiya must launch a usage-based pricing tier within Q2 to capture the price-sensitive SMB segment and increase conversion rates by 200%."

### SCQA Opening
> **Situation:** Hiya's PLG product has achieved 2M downloads with strong retention metrics.
> 
> **Complication:** However, conversion from free to paid is 0.8%—well below the 3% industry benchmark—leaving an estimated $6M annually on the table.
> 
> **Question:** How can we unlock this value without cannibalizing existing premium subscribers?
> 
> **Answer:** Introduce usage-based pricing that captures value-realized users at their moment of highest intent.

### Key Argument 1: The Conversion Gap is a Pricing Problem
- 67% of churned trials cite "price" as primary barrier (survey data)
- Users who experience >20 blocked calls convert at 4x the rate of average users
- Current pricing is binary (free vs $9.99)—no path for moderate users
- **So What:** A mid-tier option would capture the "convinced but cost-sensitive" segment

### Key Argument 2: Usage-Based Pricing Aligns Value with Revenue
- Successful PLG companies (Twilio, AWS, Zapier) use consumption models
- Our data shows clear value inflection at 50 blocked calls/month
- Model: First 50 blocks free, then $0.10/block or $4.99/month unlimited
- **So What:** Users only pay when they've experienced value, eliminating the "will this work for me?" barrier

### Key Argument 3: Implementation Risk is Low, Upside is High
- Engineering estimate: 4 weeks to implement
- Financial model shows $2.4M incremental revenue, 12% Premium cannibalization
- A/B test framework allows controlled rollout
- **So What:** We can validate the hypothesis with minimal investment before full launch

### Call to Action
1. **Week 1-2:** Finalize pricing model with Finance
2. **Week 3-6:** Engineering build + QA
3. **Week 7-8:** 10% traffic A/B test
4. **Week 9-12:** Full rollout if metrics hit (>2% conversion, <15% cannibalization)

**Decision needed:** Approve engineering allocation for Q1 build
